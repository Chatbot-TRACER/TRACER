=== CHATBOT FUNCTIONALITY ANALYSIS ===


## FUNCTIONALITIES (Workflow Structure)
- greet_user_and_introduce_pizza_ordering_capabilities: Greets the user and informs them that the chatbot can help with ordering pizzas and drinks. | Category: Chatbot Meta
  - greet_and_prompt_pizza_order_type: Greets the user and prompts them to choose between predefined or custom pizza for their order. | Category: Order Placement | Params: [pizza_type: User's choice between predefined or custom pizza. [options: predefined, custom]]
    - prompt_for_pizza_details: Prompts the user to provide pizza details, including size (small, medium, or large) and desired toppings (cheese, mushrooms, pepper, ham, bacon, pepperoni, olives, corn, chicken). The prompt can be a first-time request or a reiteration. | Category: Order Placement | Params: [pizza_toppings: The toppings the user wants on the pizza. [options: bacon, cheese, chicken, corn, ham, mushrooms, olives, pepper, pepperoni]; pizza_size: The size of the pizza [options: large, medium, small]]
      - confirm_pizza_and_prompt_for_drinks: Confirms the pizza order (including toppings and size, if applicable) and prompts the user to specify the number and type of drinks they want to order. | Category: Order Placement | Params: [number_of_drinks: The number of drinks the user wants to order; drink_choice (coke/sprite/water): The type of drink the user wants to order]
        - provide_order_summary_and_pickup_details: Provides the order summary (including items ordered), total cost, order ID, estimated readiness time, and pickup location. | Category: Order Confirmation | Outputs: [order_summary: A summary of the items included in the order.; total_cost: The total cost of the order, including all items.; estimated_readiness_time: The estimated time until the order is ready for pickup or delivery.; pickup_location: The location (e.g., store address) where the order can be picked up.; order_id: A unique identifier for the order.]
- list_available_pizza_types: Lists the available types of pizzas. | Category: Order Placement | Outputs: [available_pizza_types: A list of available pizza types, including Margarita, Carbonara, Marinera, Hawaiian, Four cheese, Vegetarian, and Custom (where the customer chooses the ingredients).]
- request_user_rephrase: Prompts the user to rephrase their input when the chatbot fails to understand the initial query. | Category: Customer Support
  - provide_troubleshooting_link: Provides a link to a troubleshooting guide for output parsing failures. | Category: Customer Support | Outputs: [troubleshooting_link: URL to troubleshooting documentation.]
- request_language_switch: Asks the user to switch to English or Spanish when the input is in a different language. | Category: Chatbot Meta
  - decline_spanish_and_prompt_for_toppings: Acknowledges the user's question about Spanish language support, clarifies that the chatbot can only communicate in English, and prompts the user to confirm or select toppings to proceed with the pizza order. | Category: Chatbot Meta | Outputs: [available_toppings_list: cheese, mushrooms, pepper, ham, bacon, pepperoni, olives, corn, or chicken]
- explain_order_options: Explains the ordering options available: predefined or custom pizzas, followed by drinks. | Category: Order Placement
- provide_pizza_price_information: Provides price information for different pizza types and sizes, including custom pizzas with a base price and additional cost per extra ingredient. | Category: Order Placement | Outputs: [pizza_price_details: Price information for Margarita, Carbonara, Marinera, Hawaiian, Four cheese, Vegetarian, and custom pizzas with up to three ingredients, including extra ingredient pricing.]
- provide_store_hours: Provides the store's operating hours. | Category: Chatbot Meta | Outputs: [store_hours: The daily operating hours of the store]
- decline_drink_information: States that it cannot provide information about drinks and redirects the user to finalize the pizza order. | Category: Order Placement
- inform_no_delivery_and_provide_pickup: Informs the user that delivery is not available and provides the shop's pickup location. | Category: Order Placement | Outputs: [pickup_location: The address or location where the order can be picked up.]

=== CHATBOT FUNCTIONALITY ANALYSIS ===

## FUNCTIONALITIES (Workflow Structure)
- provide_menu: Shows the user available options related to queries about services offered by the Center for User Attention of Information Technologies (CAU).
- explain_ticket_creation: Provides guidance on how to create a ticket, including the requirement to log in with institutional credentials and access methods (UAM IP, university Wi-Fi, or remote VPN).
  - show_ticket_buttons: Displays buttons for the user to create or consult tickets, allowing the user to select the appropriate action related to their requests.
  - confirm_ticket_requirements: Reminds the user that they need to create a ticket for technical assistance and confirms the steps required to submit it.
- clarify_service_inquiries: Assures the user that the chatbot can assist with questions regarding services available at the Center for User Attention of Information Technologies (CAU).
- provide_password_change_instructions: Gives specific instructions on how to change the ID-UAM password, including generating an activation code and necessary links.
- direct_to_faculty_resources: Redirects users to their respective faculty or school for inquiries about admission and registration processes.
- provide_admission_links: Supplies specific links to resources related to admission for different academic programs (grados, másteres, etc.).
- explain_access_requirements: Provides information regarding the necessary conditions to access the incident management system, including the need for credentials and connection requirements.
- provide_catalog_link: Offers the user a link to consult the catalog of services offered by the CAU.
- offer_service_catalog: Suggests users consult the catalog of services offered by the Center for User Attention for additional information, enabling them to explore available services beyond their current query.
- request_clarification: Asks users to rephrase their questions when the chatbot cannot clearly understand the inquiry, which helps ensure accurate responses and keeps the interaction going.
- provide_location_details: Provides the physical address and campus details for the Center for User Attention, which helps users with in-person visits.
- connect_issues: Provides follow-up options in relation to Eduroam connectivity issues, suggesting to select an option that best fits their query or to contact CAU for further assistance.

=== CHATBOT FUNCTIONALITY ANALYSIS ===

## FUNCTIONALITIES (Workflow Structure)
- provide_options: Chatbot presents the list of available pizza options for the user to choose from.
  - gather_information: Chatbot asks the user for the size of the pizza to proceed with the order.
    - select_option: User selects a medium size for the Hawaiian pizza from the available size options. (Params: pizza_size)
      - gather_drink_information: Chatbot requests information about the number and types of drinks the user wants to order.
        - submit_data: User provides the details of the drinks they want to add to their order (one Coke and one Sprite). (Params: drinks_count, drink_types)
          - process_request: Chatbot processes the order and calculates the total cost based on the selected pizza and drinks. (Params: pizza_type, pizza_size, number_of_drinks, drink_types)
            - display_results: Chatbot displays the order summary, total cost, estimated readiness time, and shop address. (Params: order_summary, total_cost, readiness_time, shop_address, order_id)
      - process_request: Chatbot processes the order and calculates the total cost based on the selected pizza and drinks. (Params: pizza_type, pizza_size, number_of_drinks, drink_types)
        - display_results: Chatbot displays the order summary, total cost, estimated readiness time, and shop address. (Params: order_summary, total_cost, readiness_time, shop_address, order_id)
- handle_error: Chatbot indicates it cannot proceed or understand the user's last input correctly.

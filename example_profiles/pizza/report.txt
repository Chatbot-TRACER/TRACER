=== CHATBOT FUNCTIONALITY ANALYSIS ===

## FUNCTIONALITIES (Workflow Structure)
- list_predefined_pizza_options: The chatbot lists predefined pizza options available for ordering.
  - select_pizza_size: The user selects a size for the pizza they want to order (small, medium, large). (Params: size)
    - confirm_pizza_order: The chatbot confirms the order of a specific type and size of pizza. (Params: pizza_type, pizza_size)
      - request_drink_order: The chatbot asks the user how many drinks they would like to order.
        - list_drink_options: The chatbot provides a list of available drink options for the user to choose from.
          - select_drink_order: The user selects specific drinks to order based on available options. (Params: drink_type, quantity)
            - confirm_order_summary: The chatbot summarizes the details of the order including food and drink items, total cost, pickup time, and location. (Params: order_details)
              - inquire_payment_methods: The user asks the chatbot about the accepted payment methods for their order.
            - order_confirmation: The chatbot confirms the user's pizza order and asks for drink specifications. (Params: pizza_type, size)
              - confirm_drink_order: The user confirms the specific types of drinks to order. (Params: drink1, drink2)
                - finalize_order_details: The chatbot provides the total cost, estimated preparation time, and order ID after confirmation. (Params: pizza_type, drink1, drink2, total_cost, preparation_time, order_id)
                  - confirm_order: Chatbot confirms the complete order, providing details such as total cost, order readiness, and order ID.
          - order_drink: The user attempts to place an order for a specific drink (Water), but the chatbot fails to correctly parse the user's request consistently. (Params: drink_name)
      - order_confirmation: The chatbot confirms the user's pizza order and asks for drink specifications. (Params: pizza_type, size)
        - confirm_drink_order: The user confirms the specific types of drinks to order. (Params: drink1, drink2)
          - finalize_order_details: The chatbot provides the total cost, estimated preparation time, and order ID after confirmation. (Params: pizza_type, drink1, drink2, total_cost, preparation_time, order_id)
            - confirm_order: Chatbot confirms the complete order, providing details such as total cost, order readiness, and order ID.
  - select_custom_pizza_ingredients: The user can inquire about and select specific ingredients for a custom pizza.
    - customize_pizza: User indicates a desire to customize their pizza toppings. (Params: toppings)
- add_item_to_cart: User adds an item (pizza, drinks) to their order cart. (Params: item_type, item_details)
- explain_custom_pizza_option: The chatbot attempts to provide information about customizing pizza but indicates limitations on knowledge.
- handle_input_error: The chatbot repeatedly fails to understand user input regarding pizza preferences, leading to a prompt for rephrasing.
- provide_pizza_prices: The chatbot provides the price details for the various types of pizzas offered.
- provide_beverage_prices: The chatbot details the prices for different beverage options available.
